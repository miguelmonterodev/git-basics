Version Control with Git and GitHub

Git is a local tool to version control projects which are organised in repositories.

  1. Download Git and check installation running git --version in the Terminal or Command Prompt

  2. git init to initialise repository

  3. git status status of the repository

  4. git add file_name add changes to be committed

  5. git commit -m "commit comment..." snapshots of the code

  6. git config --global user.name "<username>" edit username. Same with user.email

  7. git log show all commits done by branch and username

  8. git branch -m branch_name change name of the current branch

  9. git branch -b branch_name add a new branch based on the main branch

  10. git checkout branch_name change branch to branch_name

  11. git merge branch_name merge the current branch with branch_name branch

  12. git rm file_name remove file

  13. git reset --hard HEAD~1 reset first commit from HEAD

  14. git branch -D branch_name this is a command to delete branch_name

  15. git checkout --. reverts a git add as it goes to latest commit but delete all the changes in file. If we want to keep the modifications in addition to revert added file, then git reset file

GitHub is a cloud based hosting provider for Git repositories. It allows collaboration in and contributions to Web Developer's projects

  1. git remote add origin <URL>

  2. git push origin main brings the local git repository to the remote repository in GitHub

  3. git pull to update the whole repository into the local machine (Collaborations)

  4. git clone <URL> clone a repository to the local machine

  5. A Personal Access Token is needed to push to the remote repository from local machine

  6. GitHub Contributions: Fork, change/add code, and Pull Request
